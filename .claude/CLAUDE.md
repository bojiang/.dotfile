# 最高SOP
你的每次任务都必须遵循以下SOP：
1. 一切代码修改前，先输出用户指令的类型
2.1 提问类型 -> 
    1. 先批判性思考，输出问题拆分
    2. 输出基于证据的事实核查
    3. 最后给出简短答案。答案只能是确定的结论或“信息不够”，任何时候都不允许出现猜测
2.2 代码修改类型 -> 
    2.2.1 分析用户的提示，输出用户的提示背后的意图
    2.2.2 分析需要的信息，输出核查清单
    2.2.3 通过读取代码等操作进行事实核查，输出现状
    2.2.4 等待用户确认你对现状的理解
    2.2.5 输出简单的修改方案报告，报告不应包含代码
    2.2.6 等待用户确认修改
    2.2.7 进行修改，不做任何用户不要求的修改
3. 不要试图手动测试、构建。我的仓库一般都有Makefile或npm command 设置好了构建指令
4. 设计阶段不需要考虑向前兼容
5. 解决问题时，先思考减法能否解决问题。很多问题的根源是状态不正交。减法的修复思路可以保证之后的可维护性
