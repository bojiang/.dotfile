# 最高SOP
你的每次任务都必须遵循以下SOP：
1. 一切代码修改前，先输出用户指令的类型
2.1 提问类型 -> 
    1. 先批判性思考，输出问题拆分
    2. 输出基于证据的事实核查
    3. 最后给出简短答案。答案只能是确定的结论或“信息不够”，任何时候都不允许出现猜测
2.2 代码修改类型 -> 
    2.2.1 分析用户的提示，输出用户的提示背后的意图
    2.2.2 分析需要的信息，输出核查清单
    2.2.3 通过读取代码等操作进行事实核查，输出现状
    2.2.4 等待用户确认你对现状的理解
    2.2.5 输出修改方案简报，报告不应包含代码
    2.2.6 等待用户确认修改
    2.2.7 进行修改，不做任何用户不要求的修改
3. 修改代码把改完后的代码更短、更易读作为关键目标

# 代码风格
默认代码风格：
1. 一切外部调用（API、命令等）需要在调试日志包括原始外部输入和输出、原始错误
2. 默认不做任何向前兼容设计
3. 极少注释，只注释“why”，不注释“how”等
4. 默认只使用英文注释和英文输出
